Édito 3.x: l'éditeur de contenu polyvalent


GENERAL

Édito est un éditeur de contenu puissant et simple d'emploi.

POSSIBILITES

L'objectif principal de ce module est de permettre aux utilisateurs - webmasters ou adminstrateurs de site - de mettre en ligne très simplement tous types de contenu :

    1. articles et images bien sûr (Édito s'interface avec tous les éditeurs wysiwyg disponibles pour Xoops : TinyEditor, Koivi, Inbetween, etc.)
    2. scripts (html, javascript, php)
    3. flash, musiques et vidéos (tous les formats standards sont acceptés par défaut et il est possible d'en ajouter manuellement)

De nombreuses fonctions facilitent l'emploi du module Édito, notamment :

    1. un utilitaire d'upload des media et des images
    2. le clonage du module en 1 clic (avec la possibilité d'exporter / importer les données d'un clône à l'autre
    3. la protection des media par .htaccess

Parmi les atouts d'Édito, citons quelques possibilités de paramétrages :

    1. affichage extrêmement flexible des blocs
    2. option permettant la gestion automatisée de l'url Rewriting
    3. 3 modes de gestion des metas (de la singularisation manuelle des balises meta-title, meta-description et meta-keywords à l'automatisation intégrale via la technologie metagen)
    4. choix du mode d'affichage de la page d'index (images, style news, style blog ou en tableau)

Préférences (ou Paramètres généraux du module)

Avant d'utiliser Édito, nous vous suggérons de regarder de plus près la page de paramètrage du module. C'est l'endroit où vous définirez les paramètres personnels et fonctionnels du module. Ces paramètres ont un impact direct sur la page d'index, les blocs et les paramètres généraux du site.

Ils sont classés par section, signalées entre [CROCHETS], comme suit :

~[INDEX] Bannière de l'index ou des pages : saisir dans ce champ l'url de la bannière du module (formats : jpg, gif, png et swf)

~[INDEX] Intro : texte d'introduction à afficher sur la page d'index du module

~[OPTION] Texte par défaut : texte qui apparaîtra au sommet de chaque page créée (à utiliser par exemple pour conserver / définir une mise en forme cohérente). La balise {lorem} vous permettra d'effectuer des tests de "remplissage".

~[INDEX] Pied de page : saisir le contenu du pied de page du module.

~[INDEX] Mode : sélectionner le mode d'affichage de la page d'index, du plus sommaire au plus complet :

  1. Images : simple affichage du logo des pages, sans aucun contenu.
  2. Table : affichage d'un tableau plus complet, contenant diverses informations à propos de chacune des page (affiche le contenu du bloc).
  3. News : affichage à la manière d'une page d'articles (affiche le contenu du bloc).
  4. Blog : affichage pleine page, à la manière d'un blog (affiche le contenu du bloc et le contenu de la page)

~[INDEX] Affichage de l'index : définir le nombre de colonnes d'images à afficher en page d'index (de 1 à 5 colonnes, Mode images exclusivement)

~[INDEX] Classement : fixe l'ordre dans lequel les pages sont listées (ce paramètre affecte l'ensemble du module, blocs mis à part). Remarque : si l'une des deux dernières méthodes est employée, le titre de la page se contextualise et affiche soit la date de publication, soit le nombre de lectures.

    1. Titre croissant : ordre alphabétique
    2. Le plus récent : ordre chronologique de publication
    3. Le plus lu : ordonné par popularité

~[INDEX] Nombre maximum d'articles par page : sélectionner la quantité de titres à faire figurer sur la page d'accueil (de 6 à 50). Si la quantité dépasse les 50 entrées, un menu s'affiche automatiquement au sommet de la page d'index. Remarque : ce paramètre affecte le nombre de titres affichés dans tout le module (administration, blocs et listing des titres).

~[LOGO] Affichage des images : saisir les dimensions 'largeur | hauteur' des images qui figureront sur chaque page d'images. Remarque : si les dimensions de l'image uploadée sont supérieures, le redimensionnement s'effectue automatiquement et proportionnellement ; inversement, pour une image dont les dimensions sont inférieures aux limites indiquées, la taille originale est conservée.

~[LOGO] Alignement du logo : pour chaque page, sélectionner l'alignement du logo (gauche / centre / droite) ; ce paramètre n'a pas d'incidence sur le module, les logos affichés dans les blocs ou l'affichage des media (vidéo, flash, etc.)

~[PAGE] Compteur de lectures Admin : comptabiliser ou non les clics de l'administrateur dans le nombre de visites (Non est sélectionné par défaut).

~[LOGO] Taille max. des images : saisir respectivement la largeur, la hauteur et le poids des images qui peuvent être uploadées sur le serveur (séparer chacun des paramètres par | sans espace). Remarque : les restrictions fixées par l'hébergeur sont prioritaires.

~[LOGO] Répertoire des images : indiquer le répertoire dans lequel seront uploadées les images d'illustration (par défaut "uploads/edito/images", sans "/" à la fin). Remarque : si un nouveau répertoire est indiqué dans le champ, il est automatiquement créé.

~[MEDIA] Dossier de base des médias : indiquer le répertoire dans lequel seront uploadées les media (par défaut "uploads/edito/media", sans "/" à la fin). Remarque : si un nouveau répertoire est indiqué dans le champ, il est automatiquement créé.

~[MEDIA] Téléchargeable : accorder ou non la possibilité de télécharger de nouveaux media depuis les pages media (Non est sélectionné par défaut, l'icône de téléchargement ne s'affiche pas)

~[OPTION] Accès aux groupes : Sélectionner les groupes qui auront accès aux pages (la modification de ce paramètre n’aura d’incidence que sur les nouvelles créations de page ; celles déjà existantes conserveront leur propriété initiale).

~[INDEX] Soumission de pages : Sélectionner les groupes qui seront autorisés à proposer des pages depuis le formulaire public de soumission.

~[OPTION] Afficher le bloc : Afficher ou masquer le contenu du bloc dans la page (la modification de ce paramètre n’aura d’incidence que sur les nouvelles créations de page ; celles déjà existantes conserveront leur propriété initiale).

~[OPTION] Afficher titre : Afficher ou masquer le titre de la page (la modification de ce paramètre n’aura d’incidence que sur les nouvelles créations de page ; celles déjà existantes conserveront leur propriété initiale).

~[OPTION] Afficher logos : Afficher ou masquer le logo / l’image (la modification de ce paramètre n’aura d’incidence que sur les nouvelles créations de page ; celles déjà existantes conserveront leur propriété initiale).

~[OPTION] Commentaires autorisés : Afficher ou non la fonctionnalité des commentaires (la modification de ce paramètre n’aura d’incidence que sur les nouvelles créations de page ; celles déjà existantes conserveront leur propriété initiale).

~[ADMIN] Utiliser l'Editeur Wysiwyg : sélectionner l'éditeur wysiwyg à utiliser avec Édito ; pour fonctionner les modules Koivi, FCK, Inbetween, TinyEditor doivent être installés.

~[INDEX] Afficher les icônes : afficher ou non les icônes 'populaires' et 'nouveau' sur les pages (le choix s'appliquer à pour toutes les pages, y compris l'index du module).

~[INDEX] Icône nouveau : saisir le nombre du jours durant lesquels l'icône 'nouveau' doit s'afficher sur la page (7 jours par défaut, mais il n'y a pas de limites).

~[INDEX] Icône populaire : saisir le nombre de lectures à partir duquel une page doit afficher l'icône 'populaire' (100 hits par défaut, mais il n'y a pas de limites).

~[META] Meta Generateur : Édito permet de choisir entre 3 modes de génération des méta-balises "titre" et "description". Le choix effectué ici s'applique pour toutes les pages.

  1. Auto : utilisation du script metagen pour renseigner automatiquement les méta-balises (synthèse du contenu de la page)
  2. Semi-auto : utilisation du script metagen combinée aux informations saisies (pas d'analyse du contenu de la page)
  3. Manuel : les méta-balises retenues sont uniquement celles saisies dans le formulaire de création de la page

~[META] Mots clés du module : saisir les mots clés qui s'ajouteront à ceux extraits du titre de la page.

~[META] Meta Description du module : personnalisation de la méta-balise "description" pour chaque page du module. Laisser vide pour appliquer les règles de référencement définies dans l'Admin system.

~[INDEX] Page de redirection : saisir l'url de la page (ou l'id d'une page Édito) que l'on souhaite voir s'afficher à la place de la page d'index du module.

~[PAGE] Liens de navigation : pour toutes les pages, affichage ou non d'une boîte permettant de naviguer entre les pages

  1. Aucune : ne pas afficher le système de navigation
  2. Bloc : afficher sous la forme d'un bloc un menu liste placé directement sur la page
  3. Liste : afficher la liste des titres au bas de chaque page
  4. Chemin : afficher la liste des pages sous la forme Article 1 | Article 2 | Article | ...

~[LIENS] Mode Rewriting : activation ou non de la ré-écriture automatique des url. Les sources considérées sont le titre de la page et sa balise meta-titre. Les options de formatage sont :

    1. 3 lettres minimum : les mots composés de moins de 3 lettres ne sont pas retenus dans l’url
    2. 5 lettres minimum : les mots composés de moins de 5 lettres ne sont pas retenus dans l’url
    3. Tous : aucun filtre sur la taille des mots

~[MEDIA] Affichage des Medias : sélectionner le mode d'affichage des media display a navigation box under each and every module content pages.

  1. Popup : affichage dans un pop-up
  2. Page : affichage dans une page
  3. Popup & Page : permet à l’utilisateur de choisir entre les deux affichages

~[MEDIA] Taille personnalisée : Définir la taille des media (largeur|hauteur) ; ce paramètre n’a d’incidence qu’en mode Personnalisé

~[MEDIA] Medias personnalisés : saisir les extensions des fichiers media qui devront être ouverts avec Windows Media Player

~[MEDIA] Jouer en boucle : jouer ou non les media en mode perpétuel

~[MEDIA] Taille du Media : sélectionner le format d'affichage des media (le choix s'applique aux pages media existantes et à toute nouvelle création)

  1. Aucun (conserver les dimensions originales des media)
  2. Personnalisé (afficher les media aux dimensions définies dans le champ 'Taille personnalisée')
  3. TV Petit
  4. TV Moyen
  5. TV Large
  6. Cinescope Petit
  7. Cinescope Moyen
  8. Cinescope Large

~Règles des commentaires : sélectionner les règles de commentaire du module

    1. Désactiver les commentaires
    2. Approuver automatiquement les commentaires, pour tous les utilisateurs
    3. Approuver automatiquement les commentaires, pour les membres uniquement
    4. Approbation soumis au contrôle de l'administrateur

~Autoriser les anonymes à poster des commentaires ? : Non par défaut.

Paramétrage des blocs

Édito dispose de 2 types de blocs :

    1. les blocs de menu (x2) affichent des listes de pages issues du module. Par défaut ils sont intitulés Menu 1 et Menu 2
    2. les blocs de contenu (x4) affichent le contenu spécifique d'une page. Par défaut ils sont intitulés Page 1, Aléatoire, Dernier et Populaire

Cette "profusion" de blocs autorise une multitude de combinaisons et met à disposition du webmestre un grande variété d'options.

Au moment d'éditer un bloc via Blocks/Groups permissions, utiliser les options en cliquant sur le lien Editer.

a) Les blocs de Menu

~Format d'affichage : sélectionner la manière d'afficher la liste contenue dans le bloc

    1. Menu (aucune image)
    2. Liste (aucune image)
    3. Images
    4. Etendu (images et informations)

~Taille des images : saisir les dimensions du logo qui s'affichera dans le bloc (quelque soit la taille originale du logo téléchargé sur le serveur). Cette valeur, largeur-hauteur, s'appliquera à toutes les images contenues dans le bloc. Remarque : pour les logos dont les dimensions sont inférieures à celles indiquées dans le champ, la taille originale est préservée.

~Nombre de colonnes : sélectionner le nombre de colonnes à afficher dans le bloc (entre 1 et 4). Ce paramètre ne vaut que si le format d'affichage Images est actif.

~Classement par : sélectionner le critère de classement de la liste

    1. Titre croissant : affichage dans l'ordre alphabétique
    2. Le plus récent : affichage dans l'ordre chronologique de parution
    3. Le plus lu : affichage en fonction de la popularité

~Nombre de liens à afficher : sélectionner le nombre de liens à faire apparaître dans le bloc (de 6 à 50)


b) Les blocs de contenu

~[TEXT] Taille maximum dans le bloc : saisir le nombre maximum de caractères à afficher dans le bloc

~[TEXT] Choisir une page : sélectionner la page dont le contenu doit apparaître dans le bloc

    1. Au hasard : affiche dans le bloc une page au hasard (parmi celles en ligne)
    2. Le plus récent : affiche dans le bloc la page la plus récente (la page doit être en ligne)
    3. Le plus lu : affiche dans le bloc la page la plus populaire (parmi celles en ligne)
    4. Lié à la page : affiche dans le bloc la page active à l'écran
    5. Chaque jour de l'année : affiche dans le bloc la page dont l'ID correspond au jour (exemple : le 1er février, le page ciblée sera celle ayant l'ID 32)
    6. Chaque semaine de l'année : affiche dans le bloc la page dont l'ID correspond à la semaine du calendrier
    7. Chaque jour de la semaine : affiche dans le bloc la page dont l'ID correspond au jour de la semaine (limité à une rotation de 7 articles donc)
    8. Page sépcifique : affiche une page précise parmi la liste de pages créées (qu'elle soit ou non en ligne)

~[TEXT] Afficher le titre : afficher ou non dans le bloc le titre de la page (Oui par défaut)

~[TEXT] Afficher le logo : afficher ou non le logo dans le bloc (Non par défaut)

~[MEDIA] Affichage du media :

    1. Popup : affiche les media dans un pop-up
    2. Bloc : affiche les media sans la page
    3. Les 2 : permet le choix entre les deux options précédentes

Utilitaires

Une section Utilitaires met à disposition une série d'outils pour la mise à jour et la maintenance du module.

a) Téléverser des media

Formulaire de mise à jour des media (accès direct au contenu du répertoire Dossier de base des médias permettant d'ajouter / effacer des media)

b) Cloner le module

Formulaire permettant de dupliquer d'un clic le module Édito. Concernant l'intitulé du clône, eiller à respecter les restrictions suivantes :

    1. Pas de chiffre
    2. Pas plus de 16 caractères
    3. Pas de caractères spéciaux (lettres capitales, espaces, _ et - sont autorisés)
    4. Pas d'intitulé déjà existant (le terme 'content' est également prohibé)

Une fois clôné, il convient de procéder à l'installation du nouveau module Édito. Pour mieux différencier les clônes dans l'interface d'administration, il est possible de leur associer une icône spécifique en modifiant, dans le répertoire images du module, le fichier edito_slogo.

c) Exporter SQL - Importer SQL

Les formulaires d'export et d'import SQL sont conçus pour faciliter les transferts de données entre les différents clônes.

d) Editeurs disponibles

Cette rubrique permet de faire le point sur les éditeurs opérationnels sur le site. Remarque : un clic sur le titre du module redirige sur son site de développement.

f) Protéger les media

Le formulaire permet d'indiquer les sites autorisés à contourner la protection des media et logos.
